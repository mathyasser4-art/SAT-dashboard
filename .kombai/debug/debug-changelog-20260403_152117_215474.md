### Changes Made This Session

### 1. Removed incorrect `Content-Type: application/json` header from updateQuestion API
- **File:** `SAT-dashboard-master/dashboard-main/src/api/updateQuestion.api.js`
- **Change:** Deleted the `headers` object (`{ 'Content-Type': 'application/json' }`) and removed it from the `fetch()` call options.
- **Why:** The body being sent is a `FormData` object. Setting `Content-Type: application/json` overrides the browser's automatic `multipart/form-data; boundary=...` header, causing the server to attempt JSON parsing of a multipart body — which always fails. Removing the header lets the browser set the correct Content-Type automatically, matching how `addQuestion.api.js` works.
- **Revert:** Re-add `const headers = { 'Content-Type': 'application/json' }` and add `headers,` back into the fetch options object.
