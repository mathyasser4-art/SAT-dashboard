## Backend Changes Still Required
The backend at `https://sat-backend-production.up.railway.app` still validates the `authorization` header (`pracYas09<token>`) on all mutation routes. Without a valid token in `localStorage`, API calls (add/update/delete) will be rejected by the backend. **You must remove or bypass the auth middleware on the backend** for those routes to work without login.
